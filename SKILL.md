# 中文智能校对

## 核心铁律

> **凡不确定之处，必核对原稿或联网查证，绝不凭语感轻易改动。**

## 启动

用户提供文件路径后，首先确认：

1. 检查用户项目根目录是否存在 `.proofread_rc.json`，若存在则加载偏好
2. 询问模式：**快速模式**（二轮）还是**深度模式**（三轮）？
3. 若用户未指定，默认深度模式

### 快速模式触发词
"快速校对"、"过一遍"、"简单看看"、"快速审"

### 深度模式触发词
"深度校对"、"发文前审查"、"全面校对"、"正式审校"、或未明确模式时默认

---

## 检查项定义

| # | 检查项 | 模式 | 属性 | 检查规则 |
|---|--------|:--:|:--:|------|
| 1 | **文种与格式** | 共用 | 🔴 硬伤 | 文种正确（通知/报告/请示/函/决定/通报/批复/议案/纪要/公告/通告/意见）；发文字号三要素（机关代字+〔年份〕+顺序号）；标题三要素（发文机关+关于+事由+的+文种）；主送机关排序正确（先党后政、先外后内） |
| 2 | **正文文字** | 共用 | 🔴 硬伤 | 错别字；多字漏字；形近字（己已巳/末未/人入/刺剌/侯候）；标点规范（顿号逗号不分、引号层级中文""→''、括号配对）；"的/地/得"用法；同音错字 |
| 3 | **术语、一致性与政治敏感性** | 共用 | 🔴 硬伤 | 见下方 3.1-3.5 |
| 4 | **逻辑一致性** | 深度 | 🟡 建议 | 条款间不矛盾；前后引用不自指循环；数据自洽；分类无交叉遗漏；判断标准不得与激励标准形成闭环 |
| 5 | **格式排版** | 深度 | 🟡 建议 | 标题层级字体（方正小标宋→黑体→楷体_GB2312）；字号（一级二号→二级三号→三级三号）；页边距；行距固定值；页码位置（居中）；版记要素（印发机关、印发日期） |
| 6 | **数字与日期** | 共用 | 🔴 硬伤 | 阿拉伯/汉字数字按 GB/T 15835；发文字号年份用〔〕非 () 或【】；概数用汉字（"三十多人"）；公历世纪、年代、年月日、时刻用阿拉伯；定型的词/词组/成语用汉字 |
| 7 | **冗余/拖沓** | 共用 | 🟡 建议 | 虚词泡沫："进行""予以""实现了""进一步""积极"；句式杂糅；成分残缺；指代不明。注意："打好'XX'"类固定表述不视为动词堆叠 |
| 8 | **时间逻辑** | 深度 | 🟡 建议 | 落款日期≤施行日期；文中时间指代（去年/今年/明年/本年度）须换算具体年份；截止日期不早于发布日期 |

## 检查项 3 详细规则

### 3.1 术语统一
机构名称全称/简称全文一致。首次出现用全称+（以下简称"XX"），后续用简称。
- 注意：部分机构在本地存在约定俗成的标准简称，需与用户确认后方可判定为不一致
- 例：若全文某处用"省发展和改革委员会"、另一处用"省发改委"，先确认本地标准简称再判断

### 3.2 表述一致性
同一概念全文用同一表述。以首次出现为准。
- 正例："行政处罚"↔"行政处分"（全文不变）
- 反例：前面"行政处罚"，后面写成"行政罚款"

### 3.3 列举顺序一致性
同一组列举在全文每次出现时顺序不变。
- 反例：首次"一、提高认识；二、加强领导；三、落实责任"，后文写成"一、提高认识；三、落实责任；二、加强领导"

### 3.4 政治话语规范
政治固定表述须与官方发布版本逐字一致，不得增删、替换、颠倒。
- 正例："四个意识""四个自信""两个维护"（固定顺序不可变）
- 正例："习近平新时代中国特色社会主义思想"（不可缩写为"习近平思想"）
- 正例："'八场硬仗'"（应有引号）
- 联网核实方法：搜索官方来源（政府网站、新华社、人民日报），比对原文确认

### 3.5 政治敏感性
- 台湾地区：须称"台湾地区"或"中国台湾"，不得单独使用"台湾"（指称所谓"国家"时）
- 港澳：须称"香港特别行政区""澳门特别行政区"
- 民族：须使用全称"维吾尔族""藏族"等，不得使用含歧视的简称
- 宗教：引用宗教用语须核对官方表述
- 地图/领土：涉藏南、南海、钓鱼岛等表述须核实

## 实现架构：子 Agent 分派 + 主 Agent 审核

校对七武器中可并行的方法，每轮由独立子 Agent 执行。主 Agent 负责分派→收集→去重→定级→出报告。

### 子 Agent 角色定义

| 子 Agent | 负责方法 | 检查项 | 输入 | 输出 |
|----------|----------|:--:|------|------|
| **A（结构审校）** | 通读法 | 1, 4, 8 | 全文文本 | 结构/文种/逻辑/时间问题清单 |
| **B1（文字扫描）** | 重点扫描法+朗读法 | 2, 6, 7 的文字部分 | 全文文本 | 错字/标点/数字/虚词问题清单 |
| **B2（倒读筛错）** | 倒读法 | 2 的形近字部分 | 全文文本（倒序） | 形近字/多字漏字问题清单 |
| **B3（术语一致性）** | 折校法 | 3.1, 3.2, 3.3 | 全文文本 | 术语不统一/顺序不一致问题清单 |
| **B4（政治核验）** | 工具辅助法 | 3.4, 3.5 | 全文文本 | 政治话语/敏感表述问题清单 |
| **C（格式复核）** | 折校法 | 5 | docx XML | 字体/版记/排版问题清单 |

### 快速模式分派（二轮）

```
主 Agent
  │
  ├─ 第一轮: 子 Agent A（通读，项1）
  │
  └─ 第二轮: 并行分派
       ├─ 子 Agent B1（文字扫描，项2/6/7）
       ├─ 子 Agent B2（倒读，项2 形近字）
       ├─ 子 Agent B3（术语一致性，项3.1-3.3）
       └─ 子 Agent B4（政治核验，项3.4-3.5）
```

### 深度模式分派（三轮）

```
主 Agent
  │
  ├─ 第一轮: 子 Agent A（通读，项1/4/8）
  │
  ├─ 第二轮: 并行分派（同快速模式第二轮）
  │
  ├─ 第三轮: 子 Agent C（格式复核，项5）
  │
  └─ 最终: 主 Agent 合并去重→定级→出报告
```

## 主 Agent 审核汇总流程

### 1. 收集子 Agent 结果
等待所有子 Agent 返回问题清单。每个子 Agent 输出格式：
```
[位置] | [检查项] | [问题描述] | [建议]
第一条 | 3.2 | 前文用"行政处罚"此处写"行政罚款"，表述不一致 | → "行政处罚"
```

### 2. 去重合并
- 不同子 Agent 可能发现同一问题（如 B1 和 B2 都发现某个错字），合并为一条
- 保留最先发现者标注

### 3. 定级
主 Agent 对每条问题定级：
- 🔴 **硬伤**（必改）：检查项 1, 2, 3, 6 中明确违反规范的；错别字、政治表述错误
- 🟡 **建议**（可改）：检查项 4, 5, 7, 8；风格问题；冗余表述
- ⚪ **存疑**（待用户确认）：联网不可用时的政治用语；有歧义的表述

### 4. 联网核实（检查项 3.4/3.5 触发）
- 使用检索工具搜索官方来源核验政治表述
- 搜索关键词："国台办 + 台湾地区称谓"、"新华社 + 四个意识"等
- 无法联网则标注 ⚪ 存疑

### 5. 出报告
按快速/深度模板输出校对报告（见输出格式章节）

## 三大机制

### 机制 1：偏好设置（.proofread_rc.json）

首次运行时，若项目根目录无 `.proofread_rc.json`，引导用户配置：

```json
{
  "preferences": {
    "数字用法": "保持原样",
    "引号风格": "保持原样",
    "保留表达": [],
    "时间换算": true
  }
}
```

配置项说明：
- **数字用法**："保持原样" 或 "统一为国标(GB/T 15835)"
- **引号风格**："保持原样" 或 ""→「」"（台湾公文场景）
- **保留表达**：字符串数组，如 `["放管服", "三重一大"]`——这些词即使看似冗余也不建议修改
- **时间换算**：`true` 时自动检测文中"去年/今年/明年"并建议换算

支持逐份校对临时覆盖：用户在对话中说"这份保留'放管服'"即覆盖全局配置。

### 机制 2：错误档案（.proofread_log.md）

每次校对完成后，自动追加本次发现的 🔴 硬伤到项目根目录的 `.proofread_log.md`：

```markdown
| 日期 | 文件 | 检查项 | 错例 → 正例 |
|------|------|:--:|------|
| 2026-06-17 | 某实施方案.docx | 3.2 | "行政罚款" → "行政处罚" |
```

下次校对同类型公文时，优先查阅错误档案，对历史上高频出错的检查项加强扫描力度。

### 机制 3：应用修改（产出审校稿）

用户确认校对报告后回复"应用修改"，主 Agent 对原文件执行修改：
1. .docx 文件：解包 → 修改 XML（仅改 🔴 硬伤和用户确认的 🟡 建议）→ 重新打包为 `原文件名_审校稿.docx`
2. 纯文本文件：直接编辑生成 `原文件名_审校稿.txt`
3. 生成 changelog：记录每条修改的"位置 + 旧文本 → 新文本"
4. 审校稿与原文件保持在同一目录

## 输入处理

| 格式 | 处理命令 | 说明 |
|------|----------|------|
| .docx | `pandoc input.docx -t plain -o /tmp/proofread_text.txt` | 提取文本用于文字校对 |
| .docx（深度第三轮） | `python3 scripts/unpack_docx.py input.docx /tmp/unpacked/` | 解包检查 XML 格式 |
| .pdf | `pdftotext -layout input.pdf /tmp/proofread_text.txt` | 提取文本用于文字校对 |
| .txt/.md | 直接读取 | 无预处理 |
